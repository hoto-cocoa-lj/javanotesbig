hibernate实现jpa的包，如果不用jpa写代码可以不导入：
jpa是sun推出的标准，hibernate有两种操作数据库的方案，使用自己的方案(sessionfactory/session)不需要这个包，使用实现jpa的方案(entitymanagerfactory/entitymanager)需要这2个包，hibernate-entitymanager用来生成entitymanagerfactory进行各种操作/配置，spring-data-jpa用来创建JpaRepository和JpaSpecificationExecutor的子类在dao层写接口。JpaRepository用来操作数据库，JpaSpecificationExecutor用来生成Specification(spec)，spec可以加入各种条件进行查询。 

		<dependency>
			<groupId>org.hibernate</groupId>
			<artifactId>hibernate-entitymanager</artifactId>
			<version>5.0.7.Final</version>
		</dependency>
		<dependency>
			<groupId>org.springframework.data</groupId>
			<artifactId>spring-data-jpa</artifactId>
			<version>1.9.0.RELEASE</version>
		</dependency>

配置hbm映射文件，配置规则在hibernate-core包的org.hibernate最下面的
hibernate-mapping-3.0.dtd。实际项目复制粘贴即可，样本就在当前文件夹下。
配置hibernate.cfg.xml。(测试时狂报错找不到javassist，重新安装到本地就好了)

测试：
	public void test1(){
			//把hibernatec.cfg.xml放在src/main/resource下即可
		Configuration configuration = new Configuration();
			//源码可以看见configure()加载了"hibernate.cfg.xml"
		configuration.configure();
		SessionFactory sessionFactory = configuration.buildSessionFactory();
		Session se = sessionFactory.openSession();
		Transaction tx = se.beginTransaction();
		Customer c = se.get(Customer.class, 1L);
		c.setCust_name("name222");
		c.setCust_phone("222");
			//save方法返回保存的id值
		Serializable save = se.save(c);
		System.out.println(save);
		tx.commit();
		se.close();
	}
	
	
半xml半注解+jpa配置：
注解部分似乎必须用entitymanager这个包读取，且此时hibernate.cfg.xml
里没有配置mapping，所以必须配置jpa。增删改查等语法见注解加jpa的代码部分。
使用半xml半注解不需要hibernate.cfg.xml和**.hbm.xml，后者用类里的注解代替，
前者用persistence.xml代替，此文件必须放在META-INF文件夹里，
使用maven配置时放在/src/main/resources/META-INF/persistence.xml。
